# llms.txt 자동 업데이트 Skill

이 skill은 프로젝트의 현재 상태를 분석하여 llms.txt를 자동으로 업데이트합니다.

## 실행 시점

다음 경우에 이 skill을 자동으로 실행해야 합니다:
- 설정 파일 수정 후
- 새 스크립트 추가 후
- Brewfile 변경 후
- 프로젝트 구조 변경 후

## 실행 단계

1. **파일 스캔**:
   - 모든 dot_* 파일 목록
   - 모든 run_once_*.sh 스크립트
   - Brewfile, .macos 등 주요 파일
   - README.md

2. **변경 감지**:
   - git diff로 최근 변경 사항 확인
   - 새로 추가된 파일 확인
   - 삭제된 파일 확인

3. **llms.txt 업데이트**:
   - 파일 구조 섹션 업데이트
   - 핵심 구성 요소 섹션 업데이트
   - 설치 방법 검증
   - 예시 코드 업데이트

4. **검증**:
   - 마크다운 문법 체크
   - 섹션 일관성 확인
   - 중복 정보 제거

5. **완료 메시지**:
   ```
   ✅ llms.txt를 업데이트했습니다
   - 업데이트된 섹션: [섹션 목록]
   - 새로 추가된 정보: [추가 내용]
   - 변경 사항: [변경 요약]
   ```

## 사용 예시

```bash
# 명시적 실행
/update-llms

# 자동 실행 (파일 변경 후)
# → CLAUDE.md 규칙에 따라 자동 트리거
```

## 업데이트 매핑

| 변경된 파일 | 업데이트 대상 섹션 |
|------------|------------------|
| Brewfile | 핵심 구성 요소 > 패키지 관리 |
| dot_zshrc | 핵심 구성 요소 > Shell 설정 |
| dot_gitconfig.tmpl | 핵심 구성 요소 > Git 설정 |
| run_once_*.sh | 자동 설치 스크립트 + 파일 구조 |
| .macos | macOS 시스템 설정 |

## 품질 기준

- **정확성**: 모든 정보는 실제 파일 내용과 일치
- **완전성**: 주요 기능과 설정이 모두 문서화
- **명확성**: AI가 쉽게 이해할 수 있는 구조
- **최신성**: 프로젝트의 현재 상태 반영
