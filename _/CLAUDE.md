# dotfiles 프로젝트 자동화 규칙

## llms.txt 자동 업데이트 규칙

**중요**: 다음 작업을 수행할 때마다 **반드시** `llms.txt`를 자동으로 업데이트해야 합니다:

### 업데이트 트리거

1. **파일 변경 시**:
   - 새 설정 파일 추가 (dot_*, run_once_*.sh)
   - 기존 파일 수정 (Brewfile, .zshrc, .gitconfig 등)
   - 스크립트 추가 또는 삭제

2. **구조 변경 시**:
   - 새 디렉토리 생성
   - 설치 순서 변경
   - 검증 로직 추가

3. **기능 추가 시**:
   - 새 패키지 추가
   - 새 AI 도구 통합
   - 새 별칭 또는 함수 추가

### 업데이트 프로세스

작업 완료 후 다음 단계를 **자동으로** 실행:

```
1. 변경된 파일 목록 확인
2. llms.txt의 관련 섹션 식별
3. 업데이트된 내용 반영
4. 전체 문서 일관성 검증
5. llms.txt 저장
```

### 업데이트 체크리스트

- [ ] 파일 구조 섹션 업데이트
- [ ] 핵심 구성 요소 섹션 업데이트
- [ ] 설치 방법 검증
- [ ] 예시 코드 정확성 확인
- [ ] AI 어시스턴트 컨텍스트 보강

## 작업 패턴

### 패턴 1: 새 패키지 추가
```
사용자: "Brewfile에 neovim 추가해줘"
→ Brewfile 수정
→ llms.txt 자동 업데이트 (패키지 관리 섹션)
```

### 패턴 2: 스크립트 수정
```
사용자: "macOS 설정에 트랙패드 속도 추가"
→ run_once_01_macos-setting.sh 수정
→ llms.txt 자동 업데이트 (macOS 설정 섹션)
```

### 패턴 3: 새 파일 생성
```
사용자: "새로운 설치 스크립트 만들어줘"
→ run_once_10_new-script.sh 생성
→ llms.txt 자동 업데이트 (파일 구조 + 자동 설치 섹션)
```

## 실행 규칙

**CRITICAL**:
- 사용자가 "llms.txt 업데이트"를 명시적으로 요청하지 않아도
- 위의 트리거 조건에 해당하면 **무조건 자동 업데이트**
- 작업 완료 후 "llms.txt를 업데이트했습니다" 메시지 출력
